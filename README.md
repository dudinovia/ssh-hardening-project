## Описание проекта

Проект реализует максимально защищённый SSH-сервер с:
- ✅ Аутентификацией только по ключам (без паролей)
- ✅ Запретом входа под root
- ✅ Нестандартным портом
- ✅ Ограничением по пользователям/группам
- ✅ MFA (двухфакторная аутентификация) - опционально
- ✅ Отключением устаревших алгоритмов шифрования
- ✅ Автоматическим мониторингом и дашбордами

## Быстрый старт

### Предварительные требования

- Docker >= 24.0
- Docker Compose >= 2.20
- WSL2 (для Windows)

### Установка и запуск

```bash
# 1. Клонировать репозиторий
git clone https://github.com/dudinovia/ssh-hardening-project.git
cd ssh-hardening-project

# 2. Создать файл переменных окружения
cp .env.example .env
nano .env  # Отредактировать при необходимости

# 3. Запустить инфраструктуру
cd deploy
docker compose up -d --build

# 4. Проверить статус
docker compose ps
```

Для тестов:
```
pip install -r requirements.txt
pytest test_ssh_security.py -v
```
